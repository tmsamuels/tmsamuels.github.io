[Glossary Terms One]


1. Alternative Text – Alternative text is used with any images that are displayed on a site, if for some reason the image cannot be shown then the alternative text will be displayed instead. Alternative text is often called alt text and can be whatever the developer wants it to be, but in most cases it should be a description of what the image shows.


2. Attribute – The elements used in HTML have their own attributes. The attributes are values that configure the elements they are contained in, and in some cases change the way the element behaves in order to meet the user’s needs.


3. Bitmap (raster) – Bitmap graphics are made up of lots of different pixels, this is why when you zoom in on a bitmap image it goes all pixelated. GIFs are made using an array of pixels so they are considered to be bitmap, so are JPG and PNG.


4. Box Model – The box model is a type of layout in CSS, every element is represented by a box. CSS can be used to set the size, position and properties of the boxes.


5. Cascading Style Sheet (CSS) – CSS is a style sheet language that is used in conjunction with HTML, essentially the website is created using HTML but the presentation side is handled by CSS. Using CSS a plain webpage can be presented however the user likes, if they want to change the background colour or font size they can using CSS.


6. Chrome DevTools – Chrome DevTools are basically a set of debugging tools that web developers can use to take a deeper look at the inner workings of websites. Chrome DevTools can be utilised to find various errors or issues in a website and get useful information on how to optimise code.


7. Class – A class in HTML is a global attribute that can be used to give an element one or more class names. The class will keep a space separated list of these class names. An example of this would be: div class="classname", replace the “classname” with the name you wish to give the class.


8. Debugging – Debugging is the process of finding and fixing errors in HTML or CSS. There are two types of errors: syntax errors (problems with spellings) and logic errors (use of the wrong code). Generally the web developer would use tools that check the code for errors and then they would resolve the errors, but they can also try to run the code and see if it is displayed how they intended, if it isn’t then the code may need to be fixed.


9. Declaration – A declaration in CSS is a single rule that is used to specify what property of the element you wish to style. Example, h1 { color: blue; } changes the colour of the element “h1” to blue, the declaration is the “color;blue;” section.


10. Display – Display is a CSS property that identifies the type of rendering box used to display an element. The display property value can be set to none which will make it so the element is not rendered at all. There are also other display property values that make the element render in different ways.


11. Favicon – A favicon is a small icon that is displayed in the browser next to the address bar when a site is accessed. Ideally the favicon should be related to the site so that users do not get confused. Favicons can be several different image formats but the three that are most widely used are: ICO, PNG, and GIF. Favicon is short for favourite icon.


12. File path – The file path is the address that is used to locate a file, without the file path it would be impossible to find the file that you are looking for. File paths are often used when displaying images on websites, the image can only be displayed if the correct pathway to the image is provided.


13. Flex – Flex is a CSS property that dictates how an item will grow or shrink to fit within a flex container. The flex container can be any element that acts as a container for other items. The flex property allows web developers to create flexible and responsive webpages without using float or positioning.


14. Float – Float is a CSS property that is used to identify whether an element should be placed on the left or right of its container. Text will wrap around the container if it has been set to float left or right, if the container is set to float left then text will attempt to wrap to the right of the container and Vis versa.


15. Forms – A form can be created in HTML and then improved visually with CSS. A form is a collection of fields that can be used to gather information from the user. Forms are generally used on websites in the contact information page, the user can fill out the form to give the website owner information.


16. GitHub – GitHub is a hosting site that web developers use to allow others to test their site and review their code. GitHub is also used by development teams to review code and work through problems together as it has collaboration features that allow teams to share work between each other.


17. HTML5 – A mark-up language that is used to display content on a webpage. HTML is the latest version of the HTML language.

[Glossary Terms Two]


18. Hypertext – Hypertext is a form of text that uses links to take the user to other pieces of text, the links to other pieces of text are often called hyperlinks.


19. Icons – Icons are classified as pictograms that are commonly used on mobile devices instead of text. Icons are like small images that are used to replace text and can be placed anywhere on a webpage, but in most cases they are used like buttons for navigation such as the home button on most webpages.


20. JavaScript – JavaScript is a language that is used in tandem with HTML, it is mostly used on the client side but certain packages can offer server side support. JavaScript had originally been created for use a server side language by Brendan Eich. The primary use of JavaScript is to make webpages interactive for the user and provide them with online programs such as video games.


21. JQuery – jQuery is a JavaScript library that is used to make document traversal and manipulating much simpler. It also makes event handling, animation, and Ajax much easier to use. JQuery is used by lots of web developers as it simplifies and makes it easier to do some tasks.


22. Link – A link in HTML is an element that contains a relationship between the current web document and an external source. It basically creates a connection between two locations.


23. Lists – There are three main types of lists in HTML, the first is ordered lists which list content in a numbered order, the second is unordered lists which list content in bullet points, and the third is definition lists which display content with a heading and have the definition underneath. Another type of list is the nested list which is just a list within a list.



24. Media Queries – Media Queries tie into RWD, they make it so sites can adapt to different display sizes. The media query works by checking the screen size and then changing the sites layout depending on that size.


25. Navigation (nav) – Nav is an element used in HTML that’s main purpose is to provide navigation around a site or page. The nav element can be used to contain an unordered list that is filled with links to other pages or documents, then with CSS the unordered list can be presented in an appealing manner.


26. Nested elements – A nested element is an element within an element. An example of a nest element would be a link to an external site within a paragraph element on a webpage.


27. Photoshop – Photoshop is a graphics editor created by Adobe Systems, Photoshop is a raster graphics editor which means that it can be used for JPEG, PNG, GIF, and TIFF formats. Photoshop was originally conceived by Thomas and John Knoll in 1988 and has since become one of the most used and most popular raster graphics editors on the market.


28. Positioning – Positioning is used to remove elements from the layout flow of a document. Once removed these elements can be set to behave in certain ways such as sitting next to each other on the page or on top of each other. Depending on the property value that the developer specifies the behaviour of the element will change.


29. Properties (CSS) – In CSS the properties of an element are the ways that you can style it. You can choose what properties you want to alter in your rule. Example, h1 { color: blue; }, the property is the “color” section of the rule set, another potential property could be “font-size” which would alter the font size of the “h1” element.


30. Property value (CSS) – The property value is the part of the rule that actually alters the elements style, the user enters the value they want. Example, h1 { color: blue; } the property value is the “blue” section, although it could be another colour such as red or black.


31. Pseudo class – In CSS a pseudo class can be added to a selector to specify the state of the element. Basically the pseudo class can be added to a selected element to make it do something if it is in a certain state. An example of this would be: nav:hover {color: red}, the pseudo class is the “:hover” section and it essentially just changes the colour of the navigation bars text when the user hovers the mouse over it.


32. Responsive web design (RWD) – RWD is a method of web design that is aimed at making websites display properly on a wide variety of devices. A website that uses RWD will adapt to changes in the screen size to ensure that all devices can access and see the same page content.


33. Root directory – The root directory is the folder that contains all of files required for the website to function. Before building the website a suitable root directory should be established, good practice dictates that the root directory should ideally be called “htdocs”. Everything needed for the website such as images, sound files, the CSS and the HTML document should all be placed in the root directory (htdocs folder).


34. Sandbox – A sandbox is a place where you can practice and test your own code or other people’s code. The sandbox acts as an isolated testing area where code can be created and used without actually causing any real problems. Codepen.io is a great example of an online sandbox.

[Glossary Terms Three]


35. Selector – The selector in CSS is used to identify the part of the HTML that should be styled. The selector goes at the start of the rule set and is the element name from the HTML. Example HTML element: h1 … /h1, CSS selector: h1 {…}.


36. Semantic – The semantic is the meaning behind a piece of code, for example in HTML it could be what role an element has. An example of this would be the “h1” element in HTML, it gives any content within it the role of being the top heading of the page.


37. Syntax – A set of rules that dictate what symbols can and cannot be used by a language, and in some cases the combination of these symbols. Each language has its own syntax, so each symbol has a different meaning and use in different languages. Example, how to comment in HTML: !-- comment --, how to comment in CSS: /* comment */.


38. Tag/Elements – There are two types of tags in HTML, opening tags and closing tags. The opening tags go at the start of a section of content, while the closing tags go at the end. The tags essentially act like containers for the information that goes between them. Tags can sometimes be referred to as elements. Example, opening tag: h1 content goes here /h1 closing tag.


39. Text Editors – A piece of software/tool that allows the user to view and edit HTML and CSS. An example of a software based text editor would be Visual Code, while an online example would Thimble by Mozilla.


40. Transformations – Transformations in CSS are used to translate, rotate, skew, and scale elements. The transform property in CSS can be used to perform the transformations. An example of a simple transformation would be: transform: rotate(0.5turn); this would rotate the element by 180 degrees (turn it upside down).


41. Transitions – A transition in CSS can be used to make changes in property values happen smoothly. Transitions are basically used to make changes that would normally be noticeable, occur seamlessly and discreetly.


42. Universal selector – In CSS there is a selector type that can be used match any element type, the universal selector is represented by an asterisk (*) and when used it will select all elements and set properties for all of them.


43. User Interface (UI) – The UI is an interface that allows the user to interact with the website, it takes what the computer can understand and translates it into a format that the user can understand, and it also works the other way round by converting what the user inputs into a form that the computer will understand. Without the UI the user could not interact with the computer as easily.


44. Validator – A validator checks web documents to ensure that the mark-up validity is good. The validator checks the web document for errors that could potentially cause problems when trying to display the web document. When the validation process is complete, any errors will be displayed and the developer of the web document can make changes to fix the errors.


45. Vector – Vector graphics are made up of paths which means they look clearer when zoomed in on. Vectors are generally used for logos, charts, and icons among other things. Due to the fact that vector graphics are not made up of pixels, when enhanced they still retain the same sharp and clear edging they did before being zoomed in on.


46. Version control – Version control is a system that can be used to records any changes to a file over a period of time and then at a later date you can recall specific versions of the file, before or after a particular change had been made.


47. Visual Viewport – The layout viewport is a virtual screen, it is much larger than the screen size of the device that is viewing it. The visual viewport is the area of a webpage that can be seen on a device, the visual view port essentially puts part of the layout viewport into frame so that the user can see it on their screen.


48. W3C – A web standards body that is responsible for setting the rules that web developers must follow when using Hypertext and the Web. Also known as World Wide Web Consortium, headed by Sir Tim Berners-Lee.


49. Web Standards – The rules and guidelines that web developers must follow when creating a website. The use of standards helps to encourage developers to be consistent with the code they produce.


50. WHATWG – Another web standards body that sets standards for Hypertext and the Web. Started in the late 90s in response to the slow movement of the W3C. WHATWG is made up of developers that have worked in the industry for many years. Also known as Web Hypertext Application Working Group.

[Design Conclusion]


The next segment will draw some conclusions about the design process of my website. There were four website designs that I really loved to use and look at, each of the sites had their own positive and negative aspects so when I designed my website I decided I would try to emulate some of these aspects. Originally my design had lots of different colours, but after looking at the design of other sites I realised that it would be far better to only use a couple of colours and mainly just focus on different shades of the same colour.


My original design for the website would have been a multi-page design but after looking at some single page designs I decided that a single page design is very appealing and more of a modern concept for a website. The multi-page design has been around since websites were first developed, but the single page design has recently become very popular and for good reason, I believe it makes a site feel current. My original design featured similar navigation to my final design, but due to using a single page design I had to make the navigation bar fixed. Most of the single page designs I have seen use fixed navigation as it allows the user to access the navigation no matter where they are on the page.


The first design for the page had been created without input from my team member, the reason for this was to see what design I could create without help from others. I worked with my team member to review my first design and come up with all the pros and cons of the design, once I had the good and bad of the site I proceeded to create a second design that improved on the bad aspects. The designs were reviewed and improved until my team member and I found them to be adequate, we also looked at the website designs that inspired my designs and found what was good and bad about them, this ensured that I would only take inspiration from the good parts of the design while making sure to avoid adding any of the bad parts.


In order to improve my design I would consult my team member before adding or removing a feature from the design, by doing this I could get another opinion on the decision and avoid making a mistake. My original design was a multi-page website that used lots of different colours, but with the help of my team I was able to improve the design so that it wasn’t grounded in the past and had a more modern and up-to-date feel.

Over the past nine weeks I have been learning web development and in this section I will reflect upon my time learning HTML5 and CSS3. I went into learning about web development without having much knowledge of the topic, in school and college I had only designed websites using website creation sites such as Wix.com.

Learning HTML5 and CSS3 had been a new experience for me as I had never used a mark-up or styling language before. I found learning HTML5 to be very enjoyable and not too difficult, I believe this is due to the fact that I have some experience with programming languages namely VB6 and Python. I found that some of the skills I picked up while learning VB6 and Python were transferrable and made the process of learning HTML5 much easier.
I believe that after my first term of learning web development, I have learnt the basics of HTML5 and could now use what I have learn confidently, although I have only scratched the surface when it comes to learning HTML5.


During the course of the first term, I also started to learn CSS3 which is used in conjunction with HTML5. I found CSS3 to be quite hard to learn in comparison to HTML5, it was a lot easier to mess up the CSS code than the HTML. Trying to get the HTML to look aesthetically pleasing using the CSS was very hard, I could make it look better than black and white plain text but I didn’t have the knowledge or understanding of CSS to make it look like a professional site. It took several lines of code in the CSS to get the HTML to look even slightly appealing. By the end of the term I learnt how to use CSS3 at a basic level, I believe that I can use it to make a site look moderately appealing but I still have a lot to learn before I can use CSS effectively.


Originally I had planned to make a multi-page website but I eventually decided to make a single page site. I have found that a single page site is far better when you have less content to display and becomes quite hard to style when you have lots of content. I also found that the 1 2 3 column layout was very hard to create on a single page design. The 1 and 3 column parts were quite easy to set up but the 2 column part I found to be challenging. I believe with more experience and knowledge it would have been a lot easier to set up in a 1 2 3 column layout.


Once I had started to put content onto the site I found that the pages were far too long in height, I initially decided to set the text so that scroll bars appeared and the user could scroll through the text, but I found that it hindered navigation and looked out of place on my site. I resolved the issue in the end by reducing the font size to a more appropriate level.


Overall the past term has been very successful, I have learnt the basics of both HTML5 and CSS3. I can now create a basic site using HTML5 and can make a site look more appealing using CSS3. I have learnt a lot about website design while I was learning HTML and CSS, I now have knowledge of material design which will help me when I come to designing websites in the future.

My experience with my team member has been very enjoyable, I believe that working with another person had helped me to create a good design for my website as my teammate constantly gave me feedback about the positives and negatives of my designs. Some feedback from teammates can really help to refine designs and ensure that the end product looks attractive and works perfectly.


I have come to the realisation that a single page design is not optimal for this type of content and that a multiple page design would have been easier to style. I found it very hard to style the single page design, I believe that a multiple page design would have looked far more attractive. In future I would stick to a multiple page design if I needed to display lots of content and a single page design if I had minimal content to display.
